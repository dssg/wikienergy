Algorithms¶

The following algorithm modules work modules with the particular data types defined in the appliance module to perform particular disaggregation tasks such as:

  1. Appliance detection
  2. Appliance modeling and signal generation
  3. Appliance disaggregation

Factorial Hidden Markov Models¶

The fhmm module contains methods for using Hidden Markov Models (HMM) and Factorial Hidden Markov Models (FHMM) for appliance type modeling and sample generation. Given a set of appliances for a house with known parameters, FHMMs combine a number of HMMs in parallel to model appliance states.

Methods¶

disaggregator.fhmm.fit_instance_to_HMM(model, instance)[source]¶

Fits the given instance to the model. NaNs are turned into zeroes.

disaggregator.fhmm.fit_trace_to_HMM(model, trace)[source]¶

Fits the given trace to the model. NaNs are turned into zeroes.

disaggregator.fhmm.generate_FHMM_from_HMMs(type_models, key_for_model_name=None)[source]¶

Takes a dictionary of models, where the keys are the device type name, and generates an FHMM of these models.

disaggregator.fhmm.generate_HMMs_from_type(type, pi_prior, a_prior, mean_prior, cov_prior, key_for_model_name=None)[source]¶

Generates a dictionary of HMMs using each instance of given type. The key to the dictionary is defined by the parameter ‘key_for_model_name’ which looks at each instances metadata and uses the value from that key in order to name the model. If no key is given, the model is named based on its index.

disaggregator.fhmm.init_HMM(pi_prior, a_prior, mean_prior, cov_prior)[source]¶

Initializes a trace object from a series and a metadata dictionary. Series must be sampled at a particular sample rate pi_prior is the starting probability of the HMM a_prior is the transition matrix of the HMM mean_prior is the initial mean value of each state cov_prior is the initial covariance of each state

For an n-state HMM:

  • pi_prior is a 1-D numpy array of size n
  • a_prior is a 2-D numpy array of size n x n
  • mean_prior is an numpy array of size n
  • cov_prior is a 3-D numpy array that has been tiled into two rows, one column, and n third dimensional states.
    • ex) np.tile(1,(2,1,n)) for uniform covariance to start with.
disaggregator.fhmm.sort_learnt_parameters(startprob, means, covars, transmat)[source]¶

Sorts the learnt parameters for the HMM

disaggregator.fhmm.sort_startprob(mapping, startprob)[source]¶

Sort the startprob of the HMM according to power means; as returned by mapping

disaggregator.fhmm.sort_transition_matrix(mapping, A)[source]¶

Sorts the transition matrix of the HMM according to power means; as returned by mapping

Neural Networks¶

The nn module contains methods for using Neural networks for appliance detection.

Convolutional Neural Networks capture translation invariant features.

Sparse Coding¶

Sparse coding methods model appliances as a linear combinations of basis functions. It is termed “sparse coding” because appliances are coded using a sparse selection of the basis functions.
